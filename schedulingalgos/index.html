Algorithm:

(i) Input the number of processes required to be scheduled using FCFS, burst time for each process and its arrival time.

(ii) Sort the all given processes in ascending order according to arrival time in a ready queue.

(iii) Calculate the Finish Time, Turn Around Time and Waiting Time for each process which in turn help to calculate Average Waiting Time and Average Turn Around Time required by CPU to schedule given set of process using FCFS.

Algorithm:

(i) Sort all the process according to the arrival time.

(ii) Then select that process which has minimum arrival time and minimum Burst time.

(iii) After completion of process make a pool of process which after till the completion of previous process and select that process among the pool which is having minimum Burst time.

Algorithm:

(i) Sort the processes in increasing order of their arrival time.

(ii) Choose the process having highest Burst Time among all the processes that have arrived till that time.

(iii) Then process it for its burst time. Check if any other process arrives until this process completes execution.

(iv) Repeat the above steps until all the processes are executed.

Algorithm:

(i) Sort the processes in increasing order of their arrival time.

(ii) Choose the process having least arrival time with least Burst Time.

(iii) Process it for 1 unit. Check if any other process arrives upto that time of execution or not.

(iv) Repeat the above steps until all the processes are executed.

Algorithm:

(i) The processes are called according to their increasing arrival time.

(ii) If the new process arrived in the ready queue has a higher priority than the currently running process, then the incoming new process with higher priority is executed.

(iii) Once all the processes get available in the ready queue, the algorithm will behave as a non-preemptive priority.

Algorithm:

(i) All the processes are organized in the ready queue based on the arrival time. To execute the CPU processes, a queue based on FIFO is used.

(ii) Push the first process from ready queue for the task to be executed for the allotted time.

(iii) The process runs for the allocated time even if the process doesn't reach completion, still it is stopped by the process in queue until the arrival time for the next process is reached.

(iv) In a similar manner, another process is selected by the ready queue to execute its task.

(v) The above steps are repeated until all processes are finished.
